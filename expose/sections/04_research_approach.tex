\section{Research Approach}

\subsection{Data Collection}

We obtain the set of literature relevant to our review by searching the abstract, title and keywords of English literature in Scopus~\cite{scopus} with the search query \textit{"federated learning" AND "neural architecture search"} and include literature that describes a method for using NAS in FL. We do not include surveys as part of our review. We extend this initial set of literature by a single iteration of a snowball search. We exclude literature that does not provide sufficient methodological detail to extract challenges or techniques for using NAS in FL.

\subsection{Data Analysis}

We employ a qualitative research approach, specifically a systematic literature review. In our review, we develop a conceptual model that links violated centralised NAS assumptions to resulting challenges for using NAS in FL, and the techniques used to overcome these challenges as follows:

\begin{enumerate}
    \item \textbf{Challenges arising from Assumption Discrepancies:} We first create a list of challenges that arise when using NAS in FL as a result of the assumption discrepancies between the centralised setting, for which NAS methods tend to get developed, and the FL setting. For each item in our set of relevant literaure, we perform open coding to identify challenges, whereby we focus on the motivation of the work with respect to overcoming challenges of NAS in FL in particular. We do not code challenges which occur for reasons other than the aforementioned assumption discrepancies.

    \item \textbf{Influence of FL System Parameters on Challenges:} Next, we use open coding to identify FL system characteristics based on observations of varying characteristics in the literature. We assign a set of FL system characteristics to each item in the literature and use axial coding to record associations between a set of characteristics and the relevance of a challenge. 

    \item \textbf{Technique Conceptualisation:} Next, we extract techniques for using NAS in FL from the literature in two steps.
    \begin{enumerate}
        \item First, we perform open coding on each item in the literature to extract unrefined techniques for using NAS in FL. Any modification to a NAS method explicitly motivated by the federated setting is initially coded as a single unrefined technique.
        \item Then, we iteratively refine and merge the unrefined techniques by merging unrefined techniques with conceptually highly similar mechanisms into a single representative technique. Thus, we obtain a coherent set of mutually exclusive and collectively exhaustive techniques for using NAS in FL, whereby a single technique can be a synthesis of techniques from multiple source items of literature.
    \end{enumerate}

    \item \textbf{Techniques for Overcoming Challenges:} Finally, we use axial coding to extract the effect of a technique on overcoming a particular challenge from the items of literature used to synthesise a particular technique.
\end{enumerate}
